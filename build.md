# 虹光 LIVE 網頁切版專案說明書

## 1. 基礎開發規範
- **編碼**：UTF-8
- **語系**：zh-Hant
- **編碼格式**：所有原始碼檔案 (.html, .css, .js) 均採用 **UTF-8** 無 BOM 格式。
- **語言設定**：HTML `lang` 屬性設定為 `zh-Hant` (繁體中文)。
- **框架版本**：使用 **Bootstrap v5.3.2** 穩定版。
- **檔案載入**：不使用 npm，所有外部套件皆手動放置於 `/plugins` 資料夾。

## 2. 檔案架構說明
- `/assets/css/style.css`: 存放專案特有的排版與動畫樣式。
- `/assets/js/main.js`: 存放跑馬燈切換、選單觸發等交互邏輯。
- `/plugins/`: 存放 Bootstrap 核心 CSS/JS 以及 FontAwesome (若有)。
- **Mobile (< 992px)**: 使用 Offcanvas 側邊導覽選單。
- **Desktop (>= 992px)**: 顯示標準 Navbar 與右側搜尋框。

## 3. 特殊組件：公告跑馬燈 (Marquee)
### 運作原理：
1. **換則邏輯**：透過 `main.js` 控制 `.active` 類別切換。當類別切換時，CSS 會觸發 `translateY` 效果，使公告「由下往上」浮現。
2. **文字滾動**：當公告進入 `active` 狀態後，內部文字將觸發 `textScrollLTR` 動畫，實現「從左到右」的位移。
3. **後端套版建議**：
   - 請將公告內容迴圈輸出至 `<ul>` 標籤內的 `<li>`。
   - 確保第一個 `<li>` 預設帶有 `active` 類別，以確保載入即顯示。
 
## 4. 商家九宮格與品牌牆 (Merchant Grid)
### 排版重點：
- **比例固定**：使用 CSS `aspect-ratio: 1 / 1` 確保商家圖片在任何螢幕寬度下皆呈現正方形，避免跑版。
- **填充模式**：圖片一律設定 `object-fit: cover`，後端上稿時即便原始比例不一，也能自動填滿容器。
- **響應式斷點**：
  - 桌機版：`row-cols-md-3` (三欄式)。
  - 手機版：`row-cols-2` (兩欄式，方便使用者單手滑動)。

### 品牌輪播 (Brand Slider)：
- 目前採用原生 CSS `overflow-hidden` 結構。
- **JS 擴充建議**：若未來品牌數量眾多，建議於 `/plugins` 加入 **Slick.js** 或 **Swiper.js** 達成無縫循環效果。

## 5. 大直首頁 Banner 實作規範 (Hero Banner)
### 設計概念：
- **圖層分離**：背景圖、黑色遮罩與前端內容分屬不同圖層。遮罩強度可透過 `.banner-overlay` 的透明度隨時調整，確保任何背景圖下文字均可讀。
- **Logo 模組化**：品牌 Logo 採用 `.logo-card` 組件。
  - **增刪方式**：後端僅需在 `.brand-logo-grid` 容器內輸出 `div.logo-card` 即可完成增刪。
  - **顯示效果**：強制設定 `white-background` 與 `object-fit: contain`，確保不同比例的品牌商標（如長方形或圓形）都能優雅地在格狀中呈現。

### 互動與體驗：
- 加入了 CSS Hover 效果，當滑鼠移至品牌 Logo 時會觸發垂直位移，強化「品牌進駐」的視覺活性。

## 6. 商家 FB 最新消息實作說明
### 排版特性：
- **等高設計**：利用 Bootstrap 的 `.h-100` 類別，確保不論內容多寡，同一列的卡片底部都會對齊。
- **文字自動截斷**：CSS 導入了 `.line-clamp-3` 屬性。當 FB 摘要文字過長時，會自動在第三行結束並補上「...」，避免內容爆框影響排版美觀。
- **動態標籤**：`.news-badge-group` 容器可容納多個標籤（如：大直、永康、公告）。標籤採用絕對定位壓在圖片下方，增添設計層次感。

### 後端介接建議：
- **日期格式**：建議統一轉換為 `YYYY-MM-DD` 格式輸出。
- **互動數據**：如按讚數與分享按鈕，目前預留了圖示空間，未來可依據 API 串接真實數據。

## 7. 內頁佈局規範 (Inner Pages)
### 公告全文頁：
- **閱讀模式**：文字寬度建議限制在 `800px` 以內（使用 `.col-lg-8` 或 `.container`），避免因過寬造成閱讀疲勞。
- **內容圖標**：文章內的圖片一律使用 `.figure` 與 `.img-fluid`，並附帶 `.figure-caption` 作為圖說。

### 商家細節頁：
- **資訊優先**：右側資訊欄設有 `.sticky-top`，當使用者向下捲動閱讀商家故事時，預約按鈕與聯繫方式會隨時保留在視野內。
- **視覺一致性**：頁首 Banner 結合了品牌 Logo 與環境模糊底圖，強化品牌沉浸感。
